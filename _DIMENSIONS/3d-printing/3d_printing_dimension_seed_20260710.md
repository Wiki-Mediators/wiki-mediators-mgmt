# 3D Printing — Dimension Seed

## What this dimension is

Working notes for the operator's 3D printing: machine state, symptom
logs, troubleshooting trails, verdicts on fixes, and researched
reference material. Hobby lane: the bar for banking is "useful at the
printer," not "publication-grade."

## Machine state (operator-reported, as of 2026-07-10)

- Printer: Creality Ender 3 V2.
- Firmware: flashed to a BLTouch-capable build. NO BLTouch installed
  yet (planned later) — running without ABL.
- Z axis: stock single-Z leadscrew.
- Known concern: print-quality issues that MAY be Z-related (single Z,
  no ABL) — explicitly a hypothesis, not a diagnosis. Troubleshooting
  proceeds from observed symptoms, not from this prior.

## Working conventions for this dimension

- Symptom notes record: what the defect looks like, when in the print
  it appears, where on the bed/part, and what changed recently —
  before any cause is proposed.
- Fix attempts record: one change at a time, expected effect, observed
  result. Verdicts use plain outcome words (fixed / improved /
  no-effect / worsened), dated.
- Web-research claims carry source links; forum folklore is marked as
  such, same vendor-vs-forum rule as other research dimensions.

## Artifact policy

Notes sync; binaries stay local. STL/3MF/G-code/photos/video live in
`_DIMENSIONS/3d-printing/artifacts/` (never allow-listed; gitignore only
if a file would exceed the logger size guard), and notes reference them
by filename. A photo needed inside a synced note must be deliberately
allow-listed individually and kept small.
