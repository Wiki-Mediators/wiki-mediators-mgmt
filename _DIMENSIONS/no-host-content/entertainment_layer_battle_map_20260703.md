# Entertainment Layer — Battle Map, Semantic Zoom, Fractal

Consolidates the 2026-07-03 evening design arc so a fresh session has full
context. Supplements (does not replace) the battle-edges seed.

## 1. The framing: an entertainment layer

The stickman battle system is boxed as its OWN layer beside the existing
four — not Layer 5 of the spine, but an optional render layer over derived
truth. Same relationship the digest has to the vault: computed view, never
source. Its job is user retention through mundane data: keep the eye
interested while the operator traverses charts/status that would otherwise
be dry. Everything it shows must be ground truth from _DERIVED / roadmap /
altitude state; "silly little renditions" without backing state are out of
scope by rule.

## 3. The fractal / semantic-zoom concept (operator's insight)

Mechanism name: SEMANTIC ZOOM — representation changes meaning with scale,
not just size. The vault's decision structure is already hierarchical, so
the map goes fractal naturally:

  edge (range vs swing) → track (6.x capture family) → front (deployment /
  management / composition) → the thesis meta-battle (entry edge vs
  management structure — literally the altitude map's declared top thesis).

Zoomed in: two stickmen argue a t-stat. Zoomed out: the same aggregated
data renders as two huge faint figures contesting the project's soul.
Self-similar because the decision tree is self-similar.

Zoom-out payoff: INDECISION CLUSTERS. Calm regions = settled ground; stormy
regions = attention debt. Orientation at a glance — the digest's job done
preattentively.

## 4. Design rules (banked)

- Constant animation density PER SCREEN, not per node. Zoomed out, render
  only the 3-4 most contested aggregates flickering; each visible motion
  represents more the further out you go. Scarce motion = attention beacon
  (peripheral vision is motion-tuned); dense motion = noise/screensaver.
- Ground truth only: every battle binds to a real undecided item with a
  real trigger/score; resolution renders the edge settled and the loser
  fed to the void. History of finished battles = derived decision log.
- Interaction: hover/tap pops the ground-truth card ("pops up
  aggressively" is acceptable per operator; card content is vault fact,
  not flavor).
- Rendering wheels (from the seed, unchanged): Obsidian Canvas / d3-zoom /
  Leaflet-simple for zoom; LOD + IntersectionObserver culling; SMIL for
  ambient figures, JS 16-joint rig for close-up arenas; WebGL rejected
  until proven necessary.
- Gesture vocabulary: rig thinks in 16 joints; viewers read ~6-8 named
  gestures (idle, jab, recoil, taunt, bow, fallen). SMIL library covers
  defaults; the deriver can compose novel keyframes (text out) for unique
  moments.
