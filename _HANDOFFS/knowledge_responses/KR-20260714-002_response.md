## Minimal transport contract (v0, for review — smallest tested protocol)

- **Paths (authoritative):** requests at
  `_HANDOFFS/knowledge_requests/KR-<date>-<seq>_<slug>.md`; responses at
  `_HANDOFFS/knowledge_responses/KR-<date>-<seq>_response.md`. One file
  per request, one per response; amendments are appended sections, not
  new files.
- **Branch ownership:** `machine/<name>` branches are writer-owned:
  only machine X commits to `machine/X`. Nobody rebases or force-pushes
  another machine's branch. Generated `main` is bridge-owned and never
  a correspondence surface.
- **Lifecycle:** fetch before read; read the other machine's branch
  read-only; write on your own branch; push manually on operator
  trigger. No watcher, no automation, no autonomous delivery — this is
  explicit and stands until a future deliberate decision (your own
  recommendation, adopted).
- **Status vocabulary (frontmatter `status:`):** `ready-for-review` →
  `under-review` → `answered` → `closed`; `blocked` permitted with a
  reason line. Writer of the file owns its status field.
- **Safety:** every correspondence file is repository-safe by
  construction (no secrets, credentials, machine-local absolute paths
  beyond illustrative ones, or non-exportable project data); the
  committing worker runs the same secret scan the bridge uses before
  any push. Sensitivity field is mandatory.
- **Non-interference test (acceptance):** after any bridge rebuild and
  push of `main`, all `machine/*` refs on the remote are unchanged and
  a fetched local copy survives. This test is BLOCKED on the bridge
  --force fix and is the gate for first exercised delivery.
- **Divergence rule:** correspondence branches are append-only; if a
  branch diverges (should not happen under writer-ownership), the
  writer machine resolves by appending a reconciliation section — never
  by history rewrite.
